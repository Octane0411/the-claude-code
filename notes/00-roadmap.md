# 研究路线图

## 目标

产出一套可信的中文教程，解释 Claude Code 的结构，以及如何从零实现一个同等级别的系统。

## 建议阅读顺序

1. 整体总览：Claude Code 到底由哪些层组成
2. Agent Loop 主线：一次完整请求是怎么跑起来的
3. 上下文管理：系统提示、项目上下文、消息拼装
4. 工具系统：工具注册、schema、执行契约、tool-call 循环
5. 权限系统：风险控制、审批、命令执行边界
6. 记忆系统：短期上下文、长期记忆、持久化状态
7. Subagent 与任务系统：任务拆分、agent 生命周期、协作机制
8. CLI 入口与命令分发：用户是如何进入这套系统的
9. 终端 UI：React/Ink 界面如何挂接到运行时
10. MCP、插件、IDE bridge 与外部集成
11. 配置、遥测、产品化与工程化细节

## 输出策略

每个主题都尽量产出这几类内容：

- 值得优先阅读的源码文件
- 一段简明的架构摘要
- 关键控制流
- 最小可复刻实现策略
- 需要运行时验证的开放问题

优先级原则：

- 先产出 overview 和 agent loop 主线文档
- 再把其他模块作为主线的展开章节补齐
- 不按源码目录平均用力，优先解释系统真正运转的关键链路

## 验证策略

仅在以下场景使用 `../extracts/`：

- 验证特定版本的运行时行为
- 检查某个能力是否真实存在于已安装客户端
- 确认字符串、命令、打包模块痕迹
